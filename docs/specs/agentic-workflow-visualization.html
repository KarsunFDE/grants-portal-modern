Agentic Workflow: Human vs Programmatic vs AI

grants-portal-modern — AI orchestrator with explicit HITL gates, grounding controls, confidence thresholds, rejection paths, and data handoffs — 2026-06-09

Human Decisions (HITL)

  • Gate 1: Screening — Grants/Program Officer
    APPROVE / REJECT / RETURN_FOR_FIXES
  • Gate 2: COI Resolution — Review Lead
    RESOLVE_AND_CONTINUE / REMOVE_REVIEWER / OVERRIDE
  • Gate 3: Factor Acceptance — Human Reviewer
    ACCEPT / EDIT / REJECT
  • Gate 4: Final Award — Grants Officer
    AWARD / DO_NOT_AWARD / RETURN_TO_REVIEW
  • Amendment Review — Grants Officer
    APPROVE / REJECT / RETURN_FOR_FIXES

Programmatic Control Plane

  • Intake persist + ai_run_id generation
  • Prompt injection validation (fence + blocklist)
  • Tenant-bound retrieval + cache revalidation
  • Grounding checks: confidence + faithfulness thresholds
  • Idempotency check before every Bedrock call
  • Escalation routing + revision loop tracking
  • Append-only gate decision audit trail
  • SLA timer + auto-escalation on breach

AI Assistance Plane

  • Intake triage summary (pre-Gate 1 context)
  • Eligibility / risk narrative (Gate 1)
  • Factor narrative suggestions (Gate 3)
  • Award summary + SSDD draft (Gate 4)
  • Amendment draft generation
  • QA answers (no gate — informational only)
  • No autonomous final decisions

Grounding Thresholds (applies before every gate)

confidence_scorefaithfulness_scoreAction
≥ 0.80≥ 0.70Auto-proceed to gate owner
0.65 – 0.79≥ 0.70Flag LOW_CONFIDENCE + EscalationRecord — still advance to gate owner
< 0.65anyBLOCK — gate owner must release before AI repeats
any< 0.70BLOCK (citation mismatch — treated same as confidence < 0.65)

Revision loop cap: 3 per gate. Exceeding cap triggers supervisor override_flag requirement.
