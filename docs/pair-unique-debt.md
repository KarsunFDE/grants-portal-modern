## sec-cors-wildcard-credentials

- **Category:** security
- **Inject mode:** edit_existing
- **Location:** `services/grant-application-service/src/main/java/com/karsunfde/grantsportal/grantapplication/SecurityConfig.java`
- **Test:** `services/grant-application-service/src/test/java/com/karsunfde/grantsportal/grantapplication/CorsWildcardCredentialsDebtTest.java`
- **Difficulty:** medium
- **Scheduled unlock:** W4 (Wed — AI Security Engineering Day)
- **Surfaces as:** OWASP API01 finding. CORS `corsConfig()` bean uses
  `addAllowedOriginPattern("*")` AND `setAllowCredentials(true)` — wildcard + credentials
  is a known unsafe combination. Cross-origin attacker on `evil.example` can issue
  authenticated reads against `/api/grant-applications`.
- **Discovery path:** OPTIONS preflight test from a foreign origin reveals
  `Access-Control-Allow-Origin: *` AND `Access-Control-Allow-Credentials: true` —
  spec-violating combination.
- **Fix:** Replace `addAllowedOriginPattern("*")` with explicit allowlist
  (`setAllowedOrigins(List.of("https://grants-portal.karsun.gov", ...))`); keep
  `setAllowCredentials(true)`.
- **Why assessable:** OWASP API01 / W4 AI Security day; also relevant to LLM07 if
  the AI endpoint inherits the same config.

## rel-db-pool-size-one

- **Category:** reliability
- **Inject mode:** edit_existing
- **Location:** `services/grant-application-service/src/main/resources/application.yml`
- **Test:** `services/grant-application-service/src/test/java/com/karsunfde/grantsportal/grantapplication/DbPoolSizeOneDebtTest.java`
- **Difficulty:** medium
- **Scheduled unlock:** W5 (AIOps load-testing day)
- **Surfaces as:** `spring.datasource.hikari.maximum-pool-size: 1`. Every concurrent
  request serializes through the single connection. Under load (10 parallel reads)
  total elapsed time is multiplied by request count.
- **Discovery path:** W5 load-test reveals p95/p99 latency cliff once parallel
  requests exceed 1. Pair tails Hikari metrics; pool exhaustion is visible.
- **Fix:** Raise `maximum-pool-size` to 20 (or sized to expected concurrency); set
  `minimum-idle: 5`.
- **Why assessable:** AWS Well-Architected REL pillar; W5 AIOps load-testing.

## obs-pii-in-info-logs

- **Category:** observability
- **Inject mode:** edit_existing
- **Location:** `services/grant-application-service/src/main/java/com/karsunfde/grantsportal/grantapplication/service/GrantApplicationService.java`
  + new DTO fields in `dto/GrantApplicationCreateRequest.java`
- **Test:** `services/grant-application-service/src/test/java/com/karsunfde/grantsportal/grantapplication/PiiInInfoLogsDebtTest.java`
- **Difficulty:** easy
- **Scheduled unlock:** W5 (AIOps governance day)
- **Surfaces as:** `GrantApplicationService.create()` logs the Principal Investigator's
  full name and the last-4 of their SSN at INFO level. FedRAMP MP-6 / AU-2 violation.
  Bug_sketch field-name adaptation noted in `domain-mapping.md` (PI fields instead of
  generic applicant fields — fits grants-management's PI stakeholder).
- **Discovery path:** `grep -r "log.info.*PI\|log.info.*SSN" services/grant-application-service/`,
  or tail logs while creating a grant_application via the API.
- **Fix:** Drop name from log; replace SSN suffix with a hashed identifier:
  `log.info("Created {} for applicant_id={} (hashed)", saved.getId(), hash(applicant.getId()))`.
- **Why assessable:** FedRAMP MP-6 + AU-2; W5 AIOps governance day.

  comment + locked test make it cohort-visible)
- **Location:** `services/ai-orchestrator/app/bedrock_client.py`
- **Test:** `services/ai-orchestrator/tests/test_bedrock_streaming_unhandled_debt.py`
- **Difficulty:** medium
- **Scheduled unlock:** W3 (agentic UX day)
- **Surfaces as:** `bedrock_client.py` exposes only the blocking `invoke_model`. Long

- **Why assessable:** W3 agentic UX day; Bedrock streaming = real-world AI integration

## rel-file-upload-local-filesystem

- **Category:** reliability
- **Inject mode:** create_new (NOT in acquire-gov baseline)
- **Location:** `services/grant-application-service/src/main/java/com/karsunfde/grantsportal/grantapplication/service/AttachmentService.java`
- **Test:** `services/grant-application-service/src/test/java/com/karsunfde/grantsportal/grantapplication/FileUploadLocalFilesystemDebtTest.java`
- **Difficulty:** medium
- **Scheduled unlock:** W5 (cloud-native patterns day)
- **Surfaces as:** `AttachmentService.upload()` writes to `/tmp/grant-applications/uploads/`
  on the local container filesystem. Single-instance only; pod restart loses every
  attachment; horizontal scaling silently splits uploads across pods so GET-by-id
  randomly 404s.
- **Discovery path:** Upload a PI biosketch, restart the service, GET the attachment
  by id → 404. Or scale to 2 replicas and observe the same id resolves on only one pod.
- **Fix:** Replace `Files.copy` with S3 (MinIO locally) `putObject`. Return `s3://` URI
  instead of local path. `fetch()` reads via `getObject`.
- **Why assessable:** Cloud-native patterns; stateful-vs-stateless reliability; AWS S3
  introduction in W5.
